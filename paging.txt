페이징처리(pagination)
 -> 머터리얼 ui가 제공하는걸로 기억함

 동작구조 원리 
1. 페이지 번호 클릭시 페이지가 변경
2. 현재 페이지에는 active 디자인 적용
3. 이전 및 다음 버튼은 해당 방향으로 이동 가능한 페이지가 있을때만 표시
4. 이전 버튼 클릭시 이전 페이지 그룹의 마지막으로 이동
5. 다음 버튼 클릭시 다음 페이지 그룹의 첫번째로 이동
6. 각 페이지 번호 및 이전 다음 버튼은 링크를 사용하여 구현
마우스를 올리면 클릭시 이동할 url이 화면 하단에 표시

페이징 처리 구현 원리

데이터의 총 개수를 보관할 변수 : titalitems
페이지당 보여줄 데이터 개수를 보관할 변수 : itemcntpage
보여줄 페이지 개수  : pageCount
현재 페이지 번호 : curpage

1. 총 페이지 수 계산

데이터 총 개수 / 페이지당 보여줄 데이터 개수 
const totalPage = totalitems / itemcntpage

이전 , 다음 페이지 유무 확인
이전 curpage <=1;
다음 curpage >=totalpage

페이징 처리의 핵심

1. 가져올 데이터 수 
2. 공식에 맞춰서 어떻게 ui를 구성할 것인가 

첨부파일
 -> DB는 첨부파일의 정보만 저장
 -> 실질적인 파일을 저장하는 공간(storage)를 따로 분리해두는편
 -> 파일 자체를 사실 전송하는것이 아니라 
    1. 파일을 바이트코드로 변환
    2. 요청에 따라 해당 장소로 전송
    3. 저장시에 해당 바이트코드를 다시 알맞게 변환
